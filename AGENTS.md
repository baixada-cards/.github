# Agent instructions

## Scope

This repository defines the public GitHub profile and default community health
files for the Baixada organization. Keep its guidance broadly applicable.
Repository-specific policies belong in the affected repository and override
these defaults.

## Brand and copy

- The umbrella name is `Baixada`; never translate it.
- The game names `Truco`, `Escopa`, and `Bisca` are never translated.
- Write the solver-backed mode as `Truco · Lab`, using U+00B7 with spaces.
- Use a calm, precise, slightly editorial voice.
- Do not use exclamation marks, emoji, marketing superlatives, or corporate
  game-app idioms.
- Treat Portuguese and English as first-class languages. Translate meaning,
  rather than loosely paraphrasing it.

## Repository rules

- Do not name or link a project repository until that repository exists and
  its public scope is settled.
- Keep contribution, security, support, and conduct guidance consistent with
  one another.
- Never add personal email addresses, credentials, private infrastructure
  details, or unreleased security information.
- Direct vulnerability reports to GitHub private vulnerability reporting.
- Keep issue forms valid GitHub Issue Forms YAML.
- Do not add a universal workflow template unless it is genuinely safe across
  the organization's different languages and toolchains.
- State each policy's limits plainly. This is a volunteer project and does not
  promise response times.

Review changes as organization-wide defaults: a small edit here may affect
every repository that has not supplied an override.
